## Overview

Rosbridge provides a JSON API to ROS, enabling web applications to communicate with ROS topics, services, and more through a WebSocket connection. This guide covers installation for ROS 2 only.

<Banner variant="rainbow">ROS 1 is not supported. Please use ROS 2.</Banner>

## Prerequisites

Before installing rosbridge server, ensure you have:

- A working ROS 2 installation (Humble, Jazzy, or Rolling)
- Basic command-line familiarity
- Sudo privileges for package installation

</Tab>

<Tab value="Fish">

</Tab>

### Optional: DDS Configuration

For improved performance with ROS 2, set the DDS implementation in your `.bashrc`:

```bash
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
```

## Running Rosbridge

### Launch the WebSocket Server

Launch rosbridge server for ROS 2:

```bash
ros2 launch rosbridge_server rosbridge_websocket_launch.xml
```

By default, rosbridge creates a WebSocket server on **port 9090**.

![Rosbridge server running successfully](/docs/install.png)

### Custom Port Configuration

To run rosbridge on a different port, pass arguments directly :

```bash
ros2 launch rosbridge_server rosbridge_websocket_launch.xml port:=8080
```

## Verifying the Installation

### Test Connection

Once rosbridge is running, you should see output indicating the server has started:

## Troubleshooting

### Port Already in Use

If port 9090 is already occupied, either stop the conflicting service or specify a different port using the custom configuration method above.

### Connection Refused

Ensure the rosbridge server is running and check firewall settings if connecting from a remote machine

### Module Import Errors

If you encounter Python import errors, verify your ROS 2 environment is properly sourced before launching rosbridge .
